Workflow Cheatsheet

Day-to-day: edit & preview locally

cd ~/Projects/Presentations
quarto preview                    # live-reloads in browser
quarto preview fsq-llasa-inworld/slides.qmd   # preview single deck

Add a new talk

  1. Create a new folder with slides.qmd and images/:

    mkdir -p new-talk/images

  2. Add YAML front matter in new-talk/slides.qmd:

    ---
title: "Talk Title"
author: "Stanvla"
format:
  revealjs:
    theme: simple
    slide-number: true
    width: 1920
    height: 1080
---

  3. Add a link in index.qmd:

    - [Talk Title](new-talk/slides.qmd)

Images

  • Put images in <deck>/images/ next to slides.qmd

Publish to GitHub Pages

git add .
git commit -m "describe change"
git push origin main              # backup source to main branch
quarto publish gh-pages           # renders site → pushes to gh-pages branch

Quarto handles everything: renders all decks, copies images, pushes the built site to the gh-pages branch. GitHub serves it automatically.

Your site lives at: https://stanvla.github.io/speech-presentations/ Each deck at: https://stanvla.github.io/speech-presentations/fsq-llasa-inworld/slides.html

Offline single-file backup

Add embed-resources: true under format: revealjs: in a deck’s front matter. Then:

quarto render fsq-llasa-inworld/slides.qmd

The resulting slides.html is fully self-contained (images base64-embedded). AirDrop it anywhere.

Handout mode (no animations)

quarto render fsq-llasa-inworld/slides.qmd --profile handout

Uses _quarto-handout.yml to disable incremental/chalkboard.

Export to PDF

Option A — built-in print (no extra tools):

# Open the deck in browser, then append ?print-pdf to the URL:
# http://localhost:4848/fsq-llasa-inworld/slides.html?print-pdf
# Then Ctrl/Cmd+P → Save as PDF (set margins to None, enable Background graphics)

Option B — automated via decktape (needs Node.js):

npm install -g decktape
quarto render fsq-llasa-inworld/slides.qmd
decktape reveal fsq-llasa-inworld/slides.html fsq-llasa-inworld/slides.pdf

Option C — render directly to PDF format (beamer, not revealjs):

quarto render fsq-llasa-inworld/slides.qmd --to pdf

This produces a Beamer PDF (LaTeX-based, different layout from slides). Requires a TeX installation (quarto install tinytex).

Key rules

  • Never use absolute paths for images (/Users/... or /images/... breaks on Pages)
  • Don’t commit build artifacts — _site/, slides_files/, .quarto/ are gitignored
  • One quarto publish gh-pages deploys everything (all decks, landing page, images)
